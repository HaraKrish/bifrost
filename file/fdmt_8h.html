BFstatus bfFdmtInit(BFfdmt plan, BFsize nchan, BFsize max_delay, double f0, double df, double exponent, BFspace space, void *plan_storage, BFsize *plan_storage_size)¶

bfFdmtInit initialises a FDMT plan.

Return
One of the following error codes: BF_STATUS_SUCCESS, BF_STATUS_INVALID_HANDLE, BF_STATUS_INVALID_POINTER, BF_STATUS_INVALID_SPACE, BF_STATUS_INVALID_SHAPE, BF_STATUS_INVALID_STRIDE, BF_STATUS_UNSUPPORTED_DTYPE, BF_STATUS_UNSUPPORTED_STRIDE, BF_STATUS_INVALID_ARGUMENT, BF_STATUS_INSUFFICIENT_STORAGE, BF_STATUS_MEM_OP_FAILED, BF_STATUS_DEVICE_ERROR, BF_STATUS_INTERNAL_ERROR
Note

If plan_storage == NULL and plan_storage_size == NULL, the plan will manage memory itself.

If plan_storage == NULL and plan_storage_size != NULL, the function will return the required size in *plan_storage_size and do nothing else.

If plan_storage != NULL and plan_storage_size != NULL, the function will use this user-defined memory to store the plan. The memory must exist for the lifetime of the plan.

Parameters
  • plan: The FDMT plan to intialize
  • nchan: The number of frequency channels
  • max_delay: The number of delays to compute (the delay in time-samples across the band)
  • f0: The frequency of the first channel (units must match df)
  • df: The frequency step between channels (units must match f0; may be negative)
  • exponent: The exponent of the delay as a function of frequency (e.g., -2.0 for interstellar dispersion)
  • space: The memory space in which the computation will take place
  • plan_storage: Pointer to memory storage for internal plan data
  • plan_storage_size: Pointer to the size in bytes of the memory at plan_storage

BFstatus bfFdmtExecute(BFfdmt plan, BFarray const *in, BFarray const *out, BFbool negative_delays, void *exec_storage, BFsize *exec_storage_size)¶

bfFdmtExecute executes a FDMT plan.

Return
One of the following error codes: BF_STATUS_SUCCESS, BF_STATUS_INVALID_HANDLE, BF_STATUS_INVALID_POINTER, BF_STATUS_INVALID_SPACE, BF_STATUS_INVALID_SHAPE, BF_STATUS_INVALID_STRIDE, BF_STATUS_UNSUPPORTED_DTYPE, BF_STATUS_UNSUPPORTED_STRIDE, BF_STATUS_INVALID_ARGUMENT, BF_STATUS_INSUFFICIENT_STORAGE, BF_STATUS_MEM_OP_FAILED, BF_STATUS_DEVICE_ERROR, BF_STATUS_INTERNAL_ERROR
Note

If exec_storage == NULL and exec_storage_size == NULL, the function will manage memory itself and execute the plan.

If exec_storage == NULL and exec_storage_size != NULL, the function will return the required size in *exec_storage_size and do nothing else.

If exec_storage != NULL and exec_storage_size != NULL, the function will use this user-defined memory to execute the plan. The memory need exist only until the function returns.

The output TOA is aligned with the input TOA at the highest frequency, and the last (or first, if negative_delays is true) max_delay time samples of the computed output array will be incomplete.

Parameters
  • plan: The FDMT plan to execute
  • in: The input filterbank of shape [..., ntime, nchan]
  • out: The output dispersion bank of shape [..., ntime, max_delay]
  • negative_delays: If true, the function computes delays in the range (-max_delay, 0] instead of [0, max_delay)
  • exec_storage: Pointer to memory storage for temporary execution data
  • exec_storage_size: Pointer to the size in bytes of the memory at exec_storage
